# FlickBox Technologies - GitHub Organization Policy

## Access Control
- Only organization owners can invite members.
- Repository permissions must be managed through teams.

## Repository Management
- All repositories must contain README.md.
- Naming must follow the format: project-name-service.

## Branch Strategy
- main branch is production.
- No direct commits to main.
- All changes must go through pull requests.

## Code Review
- At least one approval is required before merging.

## Security
- Secrets must never be committed to repositories.
- All members must enable 2FA.

## CI/CD
- All pull requests must pass automated checks before merging.
